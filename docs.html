ColorHarmony Pro API

Documentation technique complète du générateur de palettes chromatiques basé sur la règle 60-30-10 et les principes mathématiques de la théorie des couleurs.

01 Introduction

ColorHarmony Pro est un moteur de génération de palettes chromatiques qui remplace les approches aléatoires par des algorithmes mathématiques stricts basés sur l'espace colorimétrique HSL (Hue, Saturation, Lightness).

Règle 60-30-10

Principe fondamental du design d'interface : 60% de couleur dominante (neutre), 30% de couleur secondaire (support), 10% d'accent (call-to-action).

Caractéristiques techniques

  • Calculs modulo 360° pour la roue chromatique
  • Contraintes de saturation [10%, 90%] et luminance [15%, 95%]
  • Calcul de luminance relative WCAG 2.1 exact
  • Fallback presse-papier sécurisé pour contextes non-HTTPS
  • Système de verrouillage de couleurs avec recalcul harmonique

03 Color Harmonies

Les harmonies chromatiques sont des relations angulaires définies sur la roue des couleurs (360°). Chaque algorithme produit trois valeurs HSL contraintes pour respecter la règle 60-30-10.

Algorithme Formule angulaire Caractéristique Use Case
analogous +30°, +60° Harmonie douce, faible contraste Interfaces élégantes, luxe
complementary +180° Contraste maximal CTA, alertes, sport
splitComplementary +150°, +210° Contraste adouci, triangle UI équilibrée (défaut)
monochromatic 0° (fixe) Teinte unique, variations L/S Minimalisme, SaaS

Analogous

analogous(baseHue: number): Palette
Implementation 
const analogous = (baseHue) => ({
 dominant:  { h: baseHue, s: 15, l: 95 },           // Base très claire
 secondary: { h: (baseHue + 30) % 360, s: 35, l: 40 },  // Décalage 30°
 accent:    { h: (baseHue + 60) % 360, s: 85, l: 50 }   // Décalage 60°
});

Exemple: Base 300° (Magenta) → 300°, 330°, 360°

Complementary

complementary(baseHue: number): Palette
Implementation 
const complementary = (baseHue) => ({
 dominant:  { h: baseHue, s: 20, l: 90 },
 secondary: { h: baseHue, s: 40, l: 25 },              // Même teinte, contraste L
 accent:    { h: (baseHue + 180) % 360, s: 90, l: 50 } // Opposé exact 180°
});
Attention: Le contraste maximal peut être agressif visuellement. Utiliser avec parcimonie pour les éléments d'alerte ou CTA primaires.

Split-Complementary

splitComplementary(baseHue: number): Palette
Implementation 
const splitComplementary = (baseHue) => ({
 dominant:  { h: baseHue, s: 15, l: 92 },
 secondary: { h: (baseHue + 150) % 360, s: 50, l: 45 }, // -30° de l'opposé
 accent:    { h: (baseHue + 210) % 360, s: 85, l: 55 }  // +30° de l'opposé
});
Recommandé: C'est l'algorithme par défaut car il offre le meilleur équilibre entre contraste et harmonie pour les interfaces utilisateur.

Monochromatic

monochromatic(baseHue: number): Palette
Implementation 
const monochromatic = (baseHue) => ({
 dominant:  { h: baseHue, s: 10, l: 96 },  // Très désaturé, très clair
 secondary: { h: baseHue, s: 25, l: 75 },  // Saturation moyenne
 accent:    { h: baseHue, s: 80, l: 35 }   // Forte saturation, sombre
});

04 Core API Reference

hslToHex()

hslToHex(h: number, s: number, l: number): string

Convertit une couleur HSL en notation hexadécimale #RRGGBB.

Required Number
h

Teinte 0-360°

Required Number
s

Saturation 0-100%

Required Number
l

Luminance 0-100%

Example 
hslToHex(210, 50, 50); // Returns "#4080BF"

getRelativeLuminance()

getRelativeLuminance(hex: string): number

Calcule la luminance relative d'une couleur selon la spécification WCAG 2.1. Formule: L = 0.2126×R + 0.7152×G + 0.0722×B où R, G, B sont linéarisés.

Implementation WCAG 2.1 
function getRelativeLuminance(hex) {
 const rgb = hex.replace('#', '').match(/.{2}/g).map(x => {
   const v = parseInt(x, 16) / 255;
   return v <= 0.03928 
     ? v / 12.92 
     : Math.pow((v + 0.055) / 1.055, 2.4);
 });
 return 0.2126 * rgb[0] + 0.7152 * rgb[1] + 0.0722 * rgb[2];
}

getContrastRatio()

getContrastRatio(hex1: string, hex2: string): number

Calcule le ratio de contraste entre deux couleurs. WCAG requiert ≥4.5:1 pour texte normal, ≥3:1 pour texte large.

Ratio = (L1 + 0.05) / (L2 + 0.05)

Où L1 est la luminance la plus claire et L2 la plus foncée

getAccessibleTextColor()

getAccessibleTextColor(bgHex: string): string

Détermine automatiquement si le texte doit être noir (#000000) ou blanc (#FFFFFF) pour maximiser le contraste d'accessibilité sur un arrière-plan donné.

Logic 
function getAccessibleTextColor(bgHex) {
 const whiteContrast = getContrastRatio(bgHex, '#FFFFFF');
 const blackContrast = getContrastRatio(bgHex, '#000000');
 return whiteContrast >= blackContrast ? '#FFFFFF' : '#000000';
}

05 Utilities

Clipboard Fallback

Solution de repli sécurisée pour les environnements non-HTTPS ou lorsque l'API Clipboard n'est pas disponible (anciens navigateurs, iframes, etc.).

Secure Implementation 
async function copyToClipboard(text) {
 try {
   if (navigator.clipboard && window.isSecureContext) {
     await navigator.clipboard.writeText(text);
   } else {
     // Fallback pour HTTP ou contextes non-sécurisés
     const textArea = document.createElement('textarea');
     textArea.value = text;
     textArea.style.position = 'fixed';
     textArea.style.left = '-9999px';
     document.body.appendChild(textArea);
     textArea.focus();
     textArea.select();
     document.execCommand('copy');
     document.body.removeChild(textArea);
   }
   return true;
 } catch (err) {
   console.error('Copy failed:', err);
   return false;
 }
}

06 State Management

Système de Verrouillage

Permet de figer une ou plusieurs couleurs pendant la génération. Les couleurs verrouillées servent d'ancre pour recalculer les harmonies des couleurs non-verrouillées.

Lock State & Generation Logic 
let lockedColors = { 
 primary: false, 
 secondary: false, 
 accent: false 
};

function generatePalette() {
 const harmonyType = document.getElementById('harmony-select').value;
 const baseHue = Math.floor(Math.random() * 360);
 const palette = ColorHarmonies[harmonyType](baseHue);
 
 // Ne met à jour que les couleurs non-verrouillées
 if (!lockedColors.primary) {
   currentPalette.primary = hslToHex(palette.dominant);
 }
 // ... etc
}
Comportement: Si une couleur est verrouillée, l'algorithme d'harmonie continue de générer des valeurs pour les autres couleurs, mais la teinte de base (baseHue) est recalculée aléatoirement à chaque itération. Pour une cohérence parfaite, considérez la dérivation de la teinte base à partir de la couleur verrouillée (conversion HEX→HSL inverse).
